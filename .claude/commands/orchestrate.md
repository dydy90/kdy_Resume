# Orchestrate Command

## 역할
TASKS.md 기반 개발 자동화

## 사용법
```
/orchestrate
```

## 워크플로우
1. TASKS.md 분석
2. 의존성 그래프 생성
3. 병렬 실행 가능 태스크 식별
4. 전문가 에이전트 호출
5. 진행 상황 모니터링

## 현재 프로젝트 태스크

### Phase 1: 총괄표제부 API
- P1-R1-T1: API 라우트 생성

### Phase 2: 주차 자동입력
- P2-S1-T1: 동 2개+ 시 총괄표제부 호출

### Phase 3: 건축물용도 자동입력
- P3-S1-T1: State 추가
- P3-S1-T2: 매핑 상수 정의
- P3-S1-T3: 일반상가 용도 자동입력
- P3-S1-T4: 복합/단지내 용도 자동입력
- P3-S1-T5: UI 매칭 실패 표시

### Phase 4: 층 정보 자동입력
- P4-S1-T1: State 추가
- P4-S1-T2: 총층 계산 로직
- P4-S1-T3: 해당층 자동입력
- P4-S1-T4: UI 바인딩

### Phase 5: 면적 분기 처리
- P5-S1-T1: 일반상가 면적 비활성화

### Phase 6: 검증
- P6-S1-V: 4가지 시나리오 테스트

## 병렬 실행 규칙
- P1: 독립 실행 가능
- P2~P5: 순차 실행 (같은 파일 수정)
- P6: 모든 Phase 완료 후
