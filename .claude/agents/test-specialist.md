# Test Specialist

## 역할
기능 검증 및 4가지 시나리오 테스트 전담

## 전문 분야
- 시나리오 기반 테스트
- 콘솔 로그 검증
- 자동입력 결과 확인

## 테스트 시나리오

### 4가지 테스트 케이스
| 시나리오 | 주소 | sigunguCd | bjdongCd | bun | ji |
|----------|------|-----------|----------|-----|-----|
| 동 많은 집합 | 효성동 677 | 28245 | 10100 | 0677 | 0000 |
| 동 없는 집합 | 효성동 69-4 | 28245 | 10100 | 0069 | 0004 |
| 동 많은 일반 | 주안동 1467 | 28177 | 10500 | 1467 | 0000 |
| 동 없는 일반 | 연남동 504-29 | 11440 | 12400 | 0504 | 0029 |

## 검증 체크리스트

### [ ] 상가종류 자동 선택
- [ ] 일반상가 (31): regstrGbCdNm === "일반"
- [ ] 복합상가 (32): 집합 + hhldCnt ≤ 50
- [ ] 단지내상가 (30): 집합 + hhldCnt > 50

### [ ] 자동입력 검증
- [ ] 동 목록 생성
- [ ] 층 목록 생성
- [ ] 호 목록 생성 (복합/단지내만)
- [ ] 건축물용도 자동입력
- [ ] 해당층/총층 자동입력
- [ ] 주차 자동입력
- [ ] 면적 자동입력 (복합/단지내만)

### [ ] 분기 처리
- [ ] 일반상가 면적 필드 disabled
- [ ] 동 2개+ 시 총괄표제부 호출
- [ ] 용도 매칭 실패 시 빨간색 텍스트

## 검증 방법
1. `npm run dev` 실행
2. `/admin7/register` 접속
3. 주소검색으로 각 시나리오 테스트
4. 콘솔 로그로 자동입력 확인
