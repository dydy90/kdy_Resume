# Git 백업 스킬

이 스킬은 사용자가 백업을 요청할 때 전체 git 백업 워크플로우를 자동화합니다. git 상태 확인, 변경사항 스테이징, 커밋 생성, 원격 저장소로 푸시를 처리합니다.

## 사용 시점

사용자가 다음과 같이 말할 때 이 스킬을 사용하세요:
- "백업해줘"
- "git 백업"
- "커밋하고 푸시해줘"
- "저장해줘" (git 백업 문맥에서)

## 워크플로우

다음 단계를 순서대로 실행하세요:

### 1단계: 현재 상태 확인

`git status`를 실행하여 현재 저장소 상태를 확인하세요.

### 2단계: 변경사항 스테이징

커밋되지 않은 변경사항이 있으면 `git add .`를 실행하여 모든 변경사항을 스테이징하세요.

### 3단계: 변경사항 확인 및 커밋 생성

1. `git diff --staged`를 실행하여 스테이징된 변경사항을 확인하세요
2. 변경사항을 기반으로 적절한 한글 커밋 메시지를 생성하세요:
   - 사용자가 메시지를 지정하면 그것을 사용하세요
   - 그렇지 않으면 변경사항을 기반으로 설명적인 한글 커밋 메시지를 생성하세요:
     - JSX/JS/TS 파일: "기능: 프론트엔드 컴포넌트 수정"
     - CSS 파일: "스타일: 스타일링 수정"
     - SQL 파일: "데이터베이스: 스키마 수정"
     - MD 파일: "문서: 문서 업데이트"
     - 기타: "수정: 프로젝트 파일 업데이트"
3. `git commit -m "<커밋-메시지>"`를 실행하세요

### 4단계: 원격으로 푸시

커밋이 성공적으로 생성되면 `git push`를 실행하여 원격 저장소로 변경사항을 푸시하세요.

### 5단계: 결과 보고

워크플로우가 완료되면 다음을 포함한 요약을 제공하세요:
- 변경사항에 대한 간단한 설명
- 사용된 커밋 메시지
- 푸시 완료 상태 (성공/실패)

## 백업 스크립트 사용

효율적인 실행을 위해 `scripts/git_backup.py`에 있는 번들 스크립트를 사용하세요:

```bash
python .claude/skills/git-backup/scripts/git_backup.py [커밋-메시지]
```

스크립트는 전체 워크플로우를 자동으로 처리하고 필요한 요약 정보를 반환합니다.
